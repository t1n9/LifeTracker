## 报告安全漏洞

如果您发现了安全漏洞，请不要在公开的 Issues 中报告。

### 报告方式

1. **邮件报告**（推荐）
   - 发送邮件至：[security@yourproject.com]
   - 主题：[SECURITY] LifeTracker 安全漏洞报告

2. **私有报告**
   - 使用 GitHub 的私有漏洞报告功能
   - 访问：https://github.com/your-username/LifeTracker/security/advisories

### 报告内容

请在报告中包含以下信息：
- 漏洞类型和严重程度
- 详细的漏洞描述
- 重现步骤
- 影响范围
- 建议的修复方案（如有）

### 响应时间

- **确认收到**：24小时内
- **初步评估**：72小时内
- **详细分析**：1周内
- **修复发布**：根据严重程度，1-4周内

## 安全最佳实践

### 部署安全
- 使用 HTTPS
- 定期更新依赖
- 配置防火墙
- 使用强密码和JWT密钥
- 定期备份数据

### 开发安全
- 输入验证和清理
- SQL注入防护（使用Prisma ORM）
- XSS防护
- CSRF防护
- 安全的会话管理
