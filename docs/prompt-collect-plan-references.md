# 提示词：全网抓取学习计划参考数据

## 使用方法
将下面的提示词完整复制给强力 AI（如 Claude Opus、GPT-4o、Gemini Ultra），让它搜索并整理数据，然后将输出的 JSON 粘贴到管理后台「计划参考库 → 批量导入」。

---

## 提示词正文

你是一个专业的备考资料研究员。请在网络上搜索各类考试的备考时间规划方案，整理成结构化数据。

**需要覆盖的考试类型：**
- `civil_service`：公务员考试（国考、省考、事业单位、选调生、军队文职、警察招录、税务局招录等）
- `postgraduate`：考研（全国统考、专项院校、法硕、MBA、医学考研等）
- `cet`：英语四六级（CET-4、CET-6）
- `other`：其他（教师资格证、注册会计师CPA、司法考试法考、软考、雅思、托福、PMP、建造师等）

**需要覆盖的时间跨度：**
每种考试至少提供以下几个时间维度的方案（如果该时间维度有实际意义）：
- 7天（冲刺）
- 14天（两周强化）
- 30天（一个月）
- 60天（两个月）
- 90天（三个月）
- 180天（半年）

**每条数据的要求：**
1. `examType`：必须是以下之一：`civil_service` / `postgraduate` / `cet` / `other`
2. `name`：个性化名称，要体现考试名称+时间+特点，例如"国考行测申论90天系统备考""考研数学30天强化刷题计划"
3. `matchKeywords`：逗号分隔的关键词，用于匹配用户的考试名称，例如"国考,行测,申论,公务员"
4. `durationDays`：整数天数，只能是 7/14/30/60/90/180 之一
5. `description`：**核心内容**，详细描述每个阶段/每天/每周的具体安排。要求：
   - 有实质内容，不能是泛泛而谈
   - 按天或按周写出具体科目和任务
   - 例如："第1-2天：资料分析（基础公式+专项练习100题）；第3天：言语理解（词语辨析+语句排列）；第4天：判断推理（图形推理+逻辑推理各50题）；第5天：数量关系（公式梳理+30题精练）；第6-7天：全科套卷模拟2套+错题复盘"
6. `sourceUrl`：如果你知道该方案的来源网址（知乎专栏、粉笔、华图、中公、考虫等平台的具体页面），填写完整URL；如果不确定请留空字符串
7. `sortOrder`：同类型内按时间从短到长排序，7天=1，14天=2，30天=3，60天=4，90天=5，180天=6

**输出格式（严格 JSON，不要有任何解释文字，只输出 JSON）：**

```json
[
  {
    "examType": "civil_service",
    "name": "国考/省考行测申论7天冲刺",
    "matchKeywords": "国考,省考,行测,申论,公务员",
    "durationDays": 7,
    "description": "第1天：资料分析（基础公式梳理+100题专项）；第2天：言语理解（词语辨析50题+语句排列50题）；第3天：判断推理（图形推理50题+逻辑推理50题）；第4天：数量关系（15大题型公式+30题精练）；第5天：申论（大作文结构梳理+写作1篇）；第6天：综合套卷模拟1套（135分钟限时）；第7天：全套模拟+错题复盘，重点弱项再练30题",
    "sourceUrl": "",
    "sortOrder": 1
  }
]
```

**质量要求：**
- 每个考试类型至少输出 4 条，覆盖不同时间维度
- description 必须具体可执行，不能只写"系统学习各科目"这种废话
- 如果某个考试类型+时间维度的方案在现实中不常见（如四六级180天计划），可以跳过
- 总数量不限，越多越好，30条以上为佳
- 确保 JSON 格式严格合法，可以直接被 JSON.parse() 解析
