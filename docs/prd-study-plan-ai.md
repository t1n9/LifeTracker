# PRD：学习计划 AI 引导页面

**版本：** v0.3  
**日期：** 2026-05-02  
**状态：** 需求评审中

---

## 一、背景与目标

### 背景

现有学习计划功能（StudyPlanSidebar）已支持：多步骤向导创建、科目/章节录入、系统自动平均分配每日 slot。但存在以下问题：

1. 系统按权重平均分配，不考虑用户自己对备考节奏的判断
2. 章节预计时长需要用户手动估算，门槛高
3. 创建后没有常驻入口进行调整，缺乏"计划伴随整个备考周期"的感觉
4. 没有主动提醒用户及时安排下一周

### 目标

- 让用户用**自然语言**描述备考节奏，AI 据此生成具体日历安排
- 建立**两层编排结构**（大周期阶段 + 每日 slot），降低修改成本
- 学习计划独立成页，成为**AI 常驻陪伴**入口，用户每次打开都有有效引导
- 通过**小红点提醒**驱动用户主动维护计划

---

## 二、核心概念

### 两层编排结构

```
大周期（PhasePlan）
  └── 阶段名称：如"行测强化"
  └── 时间范围：开始日 ~ 结束日
  └── 阶段目标：本阶段要完成哪些章节/科目

小周期（DailyStudySlot，以周为展示单位）
  └── 具体到每一天：日期 · 学什么章节 · 几小时
  └── 一天可以有多条 slot（不同科目/章节）
  └── 始终保持"当前周 + 下一周"处于已排好状态
```

### 可编排范围

| 时间范围 | 类型 | 是否可编排 |
|---|---|---|
| 今天及未来 | 大周期阶段 | ✅ 可修改阶段目标、调整时间范围 |
| 今天及未来 | 小周期每日 slot | ✅ 可修改具体内容/时长/增删 |
| 昨天及之前 | 已过去的 slot | ❌ 不可编排，只读展示 |

### 章节时长估算规则

- **默认值：** 每个章节默认 2 小时
- **AI 校验：** 录入章节后可触发 AI 估算，AI 综合以下因素给出建议值：
  - 考试类型（行测/申论/考研数学等不同难度）
  - 用户每日可学时长（工作日/周末）
  - 剩余天数
  - 总章节数量
- 用户可接受 AI 建议值，也可手动覆盖
- 未填写时长时，系统以默认值（2h）参与计算

## 三、功能模块

### 3.1 创建流程（重构为 6 步）

在现有 5 步基础上新增 Step 4（阶段意图）和 Step 5（日历草稿确认）。

---

#### Step 0：考试类型选择（现有，不变）

用户从模板选择：国考/省考、考研、雅思/托福、自定义。
选择后自动预填科目和章节模板。

---

#### Step 1：考试信息（现有，不变）

填写：考试名称、考试日期。
系统展示"距今 X 天"倒计时。

---

#### Step 2：时间偏好（现有，不变）

选择：全职备考 / 在职备考。
填写：工作日每天可学 N 小时、节假日/周末可学 N 小时。

---

#### Step 3：科目与章节录入（现有，优化）

**保留现有功能：** 手动添加科目、章节，支持 OCR 上传识别。

**新增：章节时长 AI 估算**

- 用户录完章节后，点击"AI 估算时长"按钮，批量填入建议时长
- 每个章节右侧展示时长输入框，预填默认值 2h，可单独修改
- 底部动态展示：
  ```
  总预计学时：约 XX 小时
  按你的每日时间安排（工作日 Nh / 周末 Nh），需要约 X 天
  ```
- 如果总时长明显超出剩余天数，给出橙色提示："章节较多，建议精简或增加每日时长"

**章节时长不强制填写**，用户可跳过，后续 AI 生成草稿时使用默认 2h。

---

#### Step 4：阶段意图描述（新增）

**目的：** 让用户用自然语言描述备考节奏，AI 理解后生成阶段划分草稿。

**AI 开场白：**
> "距离 [考试名称] 还有 [X] 天（[日期]）。你对这段时间有什么节奏想法吗？比如哪些阶段专攻哪些科目，或者最后几天怎么冲刺？"

**典型用户输入：**
- `"前10天我想专注资料分析和数量关系，中间10天申论，最后三天做套卷"`
- `"按章节顺序来，每天2小时就行"`
- `"帮我规划一下，我不太确定"`

**AI 输出阶段草稿（列表形式展示）：**

```
阶段一  第1-10天（5/3 - 5/12）  资料分析 + 数量关系
阶段二  第11-20天（5/13-5/22）  申论
阶段三  第21-23天（5/23-5/25）  套卷冲刺
```

用户可继续对话调整（"把阶段一缩短到7天"），AI 实时更新草稿。
满意后点击"下一步"进入 Step 5。

**兜底：** 若用户说"帮我规划"，AI 根据考试类型 + 科目权重 + 剩余天数自动生成推荐方案。

---

#### Step 5：日历草稿确认（新增）

**目的：** 将阶段划分展开为具体每日安排，用户确认后正式写入数据库。

**展示方式：** 以周为单位的卡片，仅展示**当前周 + 下一周**的详细草稿，更远的阶段只展示阶段卡片（不展开每日）：

```
第 1 周（5/3 - 5/9）   阶段：资料分析 + 数量关系
────────────────────────────────────────
周一 5/3    资料分析·图表题        2h
周二 5/4    资料分析·综合题        2h
周三 5/5    数量关系·基础          1.5h
            资料分析·综合复习      0.5h   ← 一天多 slot
周四 5/6    数量关系·强化          2h
周五 5/7    资料分析·专项练习      2h
周六 5/8    数量关系·综合          2h
周日 5/9    本周复习               2h
────────────────────────────────────────

第 2 周（5/10 - 5/16）  阶段：资料分析 + 数量关系
  ...（同上展开）

[阶段二卡片]  5/17 - 5/26  申论  ← 未展开，点击可展开
[阶段三卡片]  5/27 - 5/29  套卷冲刺  ← 未展开
```

**对话修改示例：**
- "把周三改成申论"
- "第一周每天减少到1.5小时"
- "第三阶段提前到第15天开始"
- "周末不学习"

AI 实时更新草稿展示，用户点击**"确认生成计划"**后，草稿正式写入数据库，跳转到常驻页面。

**草稿 vs 正式：**
- 草稿（isDraft=true）：不触发提醒、不计入进度统计
- 正式（isDraft=false）：计入进度、触发小红点检测

### 3.2 独立学习计划页面（新建）

**路由：** `/study-plan`（现在是空页面，本次改为完整独立页面）

**StudyPlanSidebar 保留但只读：** 侧边栏仍可在其他页面（如主页）唤起，展示以下两块内容，无任何编排操作入口：

**① 大范围安排（概览区）**
- 所有 PhasePlan 阶段列表：阶段名 · 时间范围 · 状态（进行中/待开始/已完成）
- 月维度进度条：本月目标学时 vs 已完成学时

**② 已排好的具体安排（当前周 + 下一周）**
- 每日 slot 列表：日期 · 章节 · 计划时长 · 完成状态
- 过去的 slot 灰色展示
- 底部跳转链接："前往学习计划页面编排 →"

#### AI 开场白逻辑

每次打开页面，AI 自动发一条开场消息：

| 场景（优先级从高到低） | AI 开场内容 |
|---|---|
| 本周缺少详细安排 | "本周（X/X - X/X）还没有排具体计划，要现在生成吗？" |
| 下周缺少详细安排 | "下周（X/X - X/X）还没安排，趁现在规划一下？" |
| 今日有待学习 slot | "距考试还有 X 天。今天建议学习：[章节] [N]h。要开始吗？" |

| 考试临近（≤7天） | "距考试只剩 [X] 天！今天安排了 [章节]，冲刺加油 💪" |
| 正常进度 | "本周完成 [M]h / [Z]h，进度不错。有什么需要调整的吗？" |

### 3.3 小红点提醒机制

#### 检测逻辑

前端启动时（每次 App 初始化）调用 `GET /study-plans/week-check`：

1. 是否有进行中（status=active）的学习计划
2. 本周（周一~周日）是否有 ≥1 个 isDraft=false 的 slot
3. 下一周（若考试日期未到）是否有 ≥1 个 isDraft=false 的 slot

**响应结构：**
```json
{
  "hasActivePlan": true,
  "thisWeekMissing": false,
  "nextWeekMissing": true,
  "planId": "xxx",
  "examDaysLeft": 23
}
```

#### 红点展示

- `thisWeekMissing=true` 或 `nextWeekMissing=true` → 导航栏"学习"入口显示红点
- 进入学习计划页面后，AI 开场白主动提及并询问
- 用户完成排期确认后，前端重新调用 week-check，红点消失

### 3.4 周计划 AI 展开

#### 触发方式

1. 创建流程 Step 5（首次）
2. 常驻页面对话："帮我排一下下周" / "生成本周计划"
3. 点击阶段卡片后 AI 询问确认
4. 小红点点进来后 AI 开场白引导

#### 后端处理流程

`POST /study-plans/:id/expand-week`

**请求：**
```json
{
  "weekStart": "2026-05-03",
  "phaseId": "phase-uuid",
  "userIntent": "这周多安排资料分析，数量关系放后半周",
  "isDraft": true
}
```

**处理步骤：**
1. 计算该周每天可用小时（工作日/节假日设置）
2. 读取该阶段涉及的章节及剩余时长（未完成的）
3. 调用 AI，传入：章节列表 + 每天时长 + 用户意图
4. AI 输出每天 slot 草稿（日期 · 章节ID · 小时数，一天可多条）
5. 返回草稿，不写库

`POST /study-plans/:id/confirm-week`

**请求：**
```json
{
  "weekStart": "2026-05-03",
  "slots": [
    { "date": "2026-05-03", "chapterId": "xxx", "chapterTitle": "资料分析·图表题", "plannedHours": 2 },
    { "date": "2026-05-04", "chapterId": "yyy", "chapterTitle": "数量关系·基础", "plannedHours": 1.5 }
  ]
}
```

写入 DailyStudySlot（isDraft=false），删除该周已有的草稿 slot（如有）。

## 四、数据库变更

### 新增表：PhasePlan

```prisma
model PhasePlan {
  id          String      @id @default(uuid())
  planId      String      @map("plan_id")
  plan        StudyPlan   @relation(fields: [planId], references: [id], onDelete: Cascade)
  name        String
  description String?
  startDate   DateTime    @map("start_date")
  endDate     DateTime    @map("end_date")
  sortOrder   Int         @default(0) @map("sort_order")
  createdAt   DateTime    @default(now()) @map("created_at")
  updatedAt   DateTime    @updatedAt @map("updated_at")
  dailySlots  DailyStudySlot[]

  @@index([planId])
  @@map("phase_plans")
}
```

### StudyPlan 新增关联

```prisma
phasePlans  PhasePlan[]
```

### DailyStudySlot 新增字段

```prisma
isDraft   Boolean  @default(false) @map("is_draft")
phaseId   String?  @map("phase_id")
phase     PhasePlan? @relation(fields: [phaseId], references: [id])
```

## 七、交互边界

### 修改权限矩阵

| 操作 | 今天 | 未来 slot | 未来阶段 | 过去 |
|---|---|---|---|---|
| 查看 | ✅ | ✅ | ✅ | ✅（只读） |
| 修改内容/时长 | ✅ | ✅ | ✅ | ❌ |
| 新增/删除 slot | ✅ | ✅ | - | ❌ |
| 标记完成 | ✅ | ❌ | - | ❌ |
| 标记跳过 | ✅ | ✅ | - | ❌ |
| 重新规划（清空重排） | ✅（今天起） | ✅ | ✅ | ❌ |

### 修改入口

1. **学习计划页面 AI 对话**（主入口）：自然语言修改任意未来安排
2. **主页面 AI 助手**：支持修改今日学习任务（通过 agent study 工具）
3. **日历视图操作菜单**：快速完成/跳过，"修改"选项唤起 AI 对话

### 不支持的操作

- 手动拖拽调整（AI 对话替代）
- 跨计划合并
- 修改已完成的 slot

## 八、异常处理

| 场景 | 处理 |
|---|---|
| 考试日期已过 | 计划自动标记 completed，页面展示历史视图，不显示红点 |
| 章节总时长 > 剩余可学时长 | Step 3 底部橙色提示，Step 5 草稿生成时 AI 给出警告 |
| AI 无法解析用户意图 | 回复："我没太理解，能再说具体一点吗？比如'前X天学XX，后X天学XX'" |
| 草稿生成失败（LLM 超时等） | 提示"生成失败，请重试"，保留上次安排 |
| 用户尝试修改过去 slot | AI 回复："过去的安排不能改了，你是想调整今天以后的计划吗？" |
| 无学习计划 | 学习计划页面展示引导创建入口，导航栏不显示红点 |
